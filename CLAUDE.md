# 교대근무 루틴 앱 (CLAUDE.md)

## 프로젝트 개요
교대근무자를 위한 루틴 관리 PWA. 기존 루틴/습관 앱은 '요일(월~일)' 기반이라
교대근무자에게 안 맞는다. 이 앱의 핵심 차별점은 요일이 아니라
'교대 주기의 Day'에 루틴을 연결하는 것이다.
- 기존 교대 앱 = '언제 일하는지'만 보여줌
- 일반 루틴 앱 = '오늘 뭘 할지' 챙기지만 요일 기반
- 이 앱 = 교대 주기에 루틴을 얹음 (이 틈만 깊게 판다)

## 파일 구조
- index.html 하나에 HTML/CSS/JS 모두 포함. 단일 파일 유지.
- 외부 라이브러리 없음. 순수 HTML/CSS/JS.

## 작업 방식 (반드시 지킬 것)
1. 수정 대상 파일은 그 작업에서 명시된 파일로만 한정한다.
   명시되지 않은 파일은 절대 건드리지 않는다.
2. 데이터 구조를 바꿀 때는 기존 localStorage 데이터를 보존한다.
   마이그레이션 코드를 넣고, 데이터 유실을 절대 만들지 않는다.
3. 수정 후에는 스스로 코드를 다시 읽고 점검한 뒤,
   무엇을 점검했는지 한국어로 요약 보고한다.
4. 모든 응답과 설명은 한국어로 한다.
5. 점검 후 문제가 있다면 다시 수정하고 점검한다.
6. 문제가 없다면 코드를 작성하고 결과물을 열어서 보여준다.


## 데이터 모델 (localStorage key: shiftRoutine)
- cycleDays: 주기 길이(정수, 상한 없음)
- baseDate: "YYYY-MM-DD" (Day 1에 해당하는 날짜)
- days[]: 각 Day 객체
  - dayIndex
  - shiftType: { name }   // 추후 hourlyRate, hours 확장 대비해 객체로 유지
  - items[]: 루틴 항목 배열
    - text
    - category   // 운동/식단/청소/준비/기타 중 하나, 없으면 '기타'
  - done   // 자리만 존재, 현재 UI 미노출

## 핵심 로직 (건드릴 때 주의)
- 날짜 차이는 반드시 로컬 자정 기준으로 계산한다.
  baseDate "YYYY-MM-DD"를 split해서 new Date(년, 월-1, 일)로 생성.
  new Date(문자열)을 직접 쓰지 말 것 (UTC 파싱으로 한국 시간대에서 하루 밀림).
  diff = Math.round((todayMid - baseMid) / 86400000)
  dayIndex = ((diff % cycleDays) + cycleDays) % cycleDays + 1
- 전체 보기 그리드: 7열 고정, 월~일 헤더.
  Day 1을 baseDate의 실제 요일 칸에 배치. 주기가 7의 배수가 아니면
  마지막 줄이 비는 것은 정상이므로 억지로 채우지 않는다.
- 카테고리 정의(색·아이콘)는 코드 상단 상수 한 곳에서 관리.
  거기만 고치면 전체에 반영되어야 한다.

## 화면 구성 (하단 탭 SPA)
- 오늘: 현재 Day + 그날의 근무유형·루틴 항목 표시
- 전체 보기: 7열 그리드, 오늘 칸 강조
- 설정: 주기 길이 + 시작 날짜 입력

## 확장 예정 (지금 만들지 말되, 데이터에 자리는 남겨둘 것)
- 교대 달력 뷰 / 급여 계산(shiftType에 시급·시간) /
  습관 통계(done 활용) / 사용자 카테고리 추가·삭제 /
  날짜 범위 네비게이션 / manifest·Service Worker·푸시 알림
- 이 기능들 때문에 기존 데이터 구조를 갈아엎지 않도록 설계.

## 안 만들 것 (포지션 유지)
- 또 하나의 교대 달력, 범용 습관 트래커가 되지 않도록 범위를 지킨다.
  넓히려는 유혹을 참고 '교대근무자 루틴'이라는 틈에 집중한다.
